# ProtoPlatform —— AI 原型设计 & 托管协作一体化平台

> 核心目标：让产品团队用 AI 工具输出更符合公司 UI 规范的原型，同时统一管理、预览、协作所有原型文件。

---

## 一、整体架构概览

```
ProtoPlatform
├── 模块 A：AI 原型生成平台（Design System + AI Page Gen）
│   ├── 设计稿解析 → 组件库提取
│   ├── 多组件库管理
│   └── 提示词 → 规范页面生成
│
└── 模块 B：原型托管协作平台（Proto Hub）
    ├── Axure / Sketch 插件发布
    ├── 原型预览 / 分享 / 评论
    ├── 项目文件夹管理
    └── 组件库托管（复用模块 A）
```

两个模块共用同一套 **组件库系统** 和 **用户权限体系**，对外是一个统一产品入口。

## 二、模块 A：AI 原型生成平台

### 2.1 功能清单

| 功能 | 实现方式 | 优先级 |
|------|---------|--------|
| 上传设计稿（Figma/Sketch/PNG/PDF）| 文件上传 + AI 解析 | P0 |
| AI 自动提取组件库（色值、字体、图标、间距、线条） | Vision 大模型 + CSS Token 导出 | P0 |
| 手动创建/编辑组件库 | 可视化编辑器 | P1 |
| 多组件库管理 | 组件库列表，支持版本 | P0 |
| 提示词 + 指定组件库 → 生成页面 | LLM + RAG 组件库注入 | P0 |
| 生成页面预览 / 导出 HTML | 在线渲染 | P0 |
| 弹窗选择组件库（非提示词模式）| 交互弹窗 | P1 |

### 2.2 AI 设计稿解析流程

```
用户上传文件（Figma Token / Sketch / 截图）
        ↓
文件预处理（图片 → base64 / Figma API 拉取）
        ↓
Vision LLM 分析（GPT-4V / Claude Vision）
        ↓
结构化输出（JSON Design Token）
   {
     colors: { primary: "#1677FF", ... },
     typography: { h1: { size: 24, weight: 600 }, ... },
     spacing: [4, 8, 12, 16, 24, 32],
     borderRadius: [4, 8, 12],
     icons: [ { name: "home", svg: "..." }, ... ],
     shadows: [...],
     components: [ { name: "Button", variants: [...] } ]
   }
        ↓
组件库入库 + 可视化展示
```

### 2.3 AI 页面生成流程

```
用户输入提示词："设计一个用户登录页，简洁风格"
        ↓
选择/指定组件库（弹窗 or 提示词中指定）
        ↓
构建 System Prompt（注入组件库 Token + 组件列表）
        ↓
LLM 生成 HTML/React JSX
        ↓
在线预览渲染 → 可导出 / 保存到项目
```

## 三、模块 B：原型托管协作平台

### 3.1 功能清单

| 功能 | 实现方式 | 优先级 |
|------|---------|--------|
| Axure 发布到平台 | Axure 插件（AxureRP Plugin）| P0 |
| Sketch 发布到平台 | Sketch Plugin | P1 |
| 原型在线预览（iframe 嵌入）| 静态文件托管 | P0 |
| 分享链接（含密码保护）| Token 短链 | P0 |
| 评论（支持定位到页面坐标）| Canvas 坐标评论系统 | P0 |
| 文件夹管理（产品线）| 树形目录 | P0 |
| 版本历史 | 文件版本链 | P1 |
| 组件库托管（复用模块 A）| 统一组件库系统 | P1 |
| 成员权限管理 | 角色：Owner/Editor/Viewer | P0 |

### 3.2 Axure 插件发布流程

```
产品在 Axure 中完成原型设计
        ↓
点击"发布预览"按钮 → 触发 ProtoPlatform 插件
        ↓
插件弹窗：选择目标项目 / 文件夹 / 版本说明
        ↓
Axure 生成 HTML 包 → 插件调用平台 Upload API
        ↓
平台存储静态文件（OSS/COS）
        ↓
返回预览链接 → 自动复制到剪贴板
        ↓
分享给产品/开发/设计同学在线查看
```

> **Axure 插件实现方式**：Axure RP 支持自定义「发布目标」插件（Publishing Plugin），基于 Axure Plugin API 开发，调用平台 REST API 上传打包后的 HTML 文件。

### 3.3 评论系统设计

- 评论支持两种模式：
  - **页面级评论**：对某个原型页面整体评论
  - **坐标级评论**：点击原型某个位置，插入定位标注（类似 Figma 评论）
- 评论支持 @成员、回复、已解决标记

## 四、技术选型

### 前端
| 技术 | 用途 |
|------|------|
| React 18 + TypeScript | 主框架 |
| TDesign / Ant Design | UI 组件库 |
| Vite | 构建工具 |
| Monaco Editor | 代码预览/编辑 |
| Fabric.js / Konva | 原型画布 + 坐标评论 |

### 后端
| 技术 | 用途 |
|------|------|
| Node.js + NestJS | 主服务框架 |
| PostgreSQL | 结构化数据 |
| Redis | 缓存 / Session |
| 腾讯云 COS / 阿里云 OSS | 静态文件托管 |
| OpenAI / 混元 Vision API | 设计稿解析 |

### AI 层
| 技术 | 用途 |
|------|------|
| GPT-4V / Claude Vision | 设计稿图像解析 |
| GPT-4o / DeepSeek | 页面代码生成 |
| LangChain | Prompt 编排 + RAG |
| Figma REST API | 直接拉取 Figma 设计 Token |

### 插件
| 插件 | 技术 |
|------|------|
| Axure 发布插件 | Axure Plugin API（JS） |
| Sketch 发布插件 | Sketch Plugin API（CocoaScript/JS）|

## 八、关键差异化优势

1. **AI 组件库提取** — 上传截图/设计稿自动生成符合团队规范的 Design Token，不需要手动整理
2. **规范约束的 AI 页面生成** — LLM 生成页面时注入组件库约束，输出符合公司 UI 规范，而非通用风格
3. **原型托管一体化** — 设计稿分析、原型托管、评论协作在同一平台，减少工具跳转
4. **插件无缝发布** — Axure 一键发布，产品无需手动导出上传
